## Welcome to omegaUp!

This guide will walk you through creating an account, finding a problem, and submitting your first solution. In just a few minutes, you'll be on your way to improving your programming skills.

<Steps>
  <Step title="Create your account">
    Navigate to [omegaup.com](https://omegaup.com) and click **Sign up** in the top right corner.
    
    Fill in the registration form with:
    - **Username**: Choose a unique username (3-20 characters)
    - **Email**: Your email address for account verification
    - **Password**: A secure password (at least 8 characters)
    
    <Note>
      You'll receive a verification email. Click the link to activate your account.
    </Note>
  </Step>

  <Step title="Explore the problem library">
    Once logged in, go to the **Problems** section to browse thousands of programming challenges.
    
    **Finding your first problem:**
    - Filter by difficulty: Start with "Easy" problems
    - Search by topic: Try "arrays" or "loops" for beginners
    - Sort by acceptance rate: Higher rates mean more users have solved it
    
    <Tip>
      Try problem "A+B" or "Hello World" for your first submission - they're great starting points!
    </Tip>
  </Step>

  <Step title="Read the problem statement">
    Click on a problem to see:
    - **Problem description**: What you need to solve
    - **Input format**: How data will be provided to your program
    - **Output format**: What your program should output
    - **Sample cases**: Example inputs and expected outputs
    - **Constraints**: Time and memory limits
    
    **Example problem: Sum of two numbers**
    
    ```
    Input:
    5 3
    
    Output:
    8
    ```
    
    Your program should read two integers from standard input and output their sum.
  </Step>

  <Step title="Write your solution">
    Use the built-in code editor in the arena to write your solution. Choose your preferred programming language from the dropdown.
    
    <Tabs>
      <Tab title="C++">
        ```cpp
        #include <iostream>
        using namespace std;
        
        int main() {
            int a, b;
            cin >> a >> b;
            cout << a + b << endl;
            return 0;
        }
        ```
      </Tab>
      
      <Tab title="Python">
        ```python
        a, b = map(int, input().split())
        print(a + b)
        ```
      </Tab>
      
      <Tab title="Java">
        ```java
        import java.util.Scanner;
        
        public class Main {
            public static void main(String[] args) {
                Scanner sc = new Scanner(System.in);
                int a = sc.nextInt();
                int b = sc.nextInt();
                System.out.println(a + b);
            }
        }
        ```
      </Tab>
    </Tabs>
    
    <Info>
      The editor supports syntax highlighting, autocomplete, and real-time error detection for all supported languages.
    </Info>
  </Step>

  <Step title="Submit your solution">
    When you're ready:
    1. Select your programming language from the dropdown
    2. Click the **Submit** button (or press Ctrl+S)
    3. Wait for the grading system to evaluate your code
    
    Your submission will be tested against all test cases automatically.
  </Step>

  <Step title="View your results">
    After a few seconds, you'll see your verdict:
    
    <Check>**Accepted (AC)**: Congratulations! Your solution is correct! ✓</Check>
    
    If you didn't get Accepted on your first try, don't worry! Common verdicts include:
    - **Wrong Answer (WA)**: Your output doesn't match the expected results
    - **Time Limit Exceeded (TLE)**: Your solution is too slow
    - **Compilation Error (CE)**: Your code has syntax errors
    - **Runtime Error (RTE)**: Your program crashed during execution
    
    Review the test case results to see which cases failed, then revise and resubmit.
  </Step>

  <Step title="Track your progress">
    View your profile to see:
    - Number of problems solved
    - Your current rank and score
    - Recent submissions and verdicts
    - Earning badges for achievements
    
    <Tip>
      Set a goal to solve at least one problem per day to build consistency!
    </Tip>
  </Step>
</Steps>

## Understanding verdicts

When you submit a solution, the grading system tests it against multiple test cases and returns a verdict:

| Verdict | Meaning | What to do |
|---------|---------|------------|
| **AC** | Accepted | Your solution is correct! |
| **WA** | Wrong Answer | Check your logic and output format |
| **TLE** | Time Limit Exceeded | Optimize your algorithm |
| **MLE** | Memory Limit Exceeded | Reduce memory usage |
| **RTE** | Runtime Error | Fix crashes (array bounds, null pointers, etc.) |
| **CE** | Compilation Error | Fix syntax errors |

<AccordionGroup>
  <Accordion title="How does the grading system work?">
    omegaUp uses an automated grading system that:
    
    1. **Compiles** your code (for compiled languages)
    2. **Executes** your program against each test case
    3. **Compares** your output with the expected output
    4. **Measures** runtime and memory usage
    5. **Assigns** a verdict and score
    
    All code runs in a secure sandbox with strict resource limits.
  </Accordion>

  <Accordion title="What are test cases?">
    Test cases are input files and their corresponding expected outputs. Your solution must produce the correct output for all test cases to be accepted.
    
    - **Sample cases**: Usually shown in the problem statement
    - **Hidden cases**: Not revealed to prevent hardcoding answers
    - **Edge cases**: Test boundary conditions and special scenarios
  </Accordion>

  <Accordion title="Can I see the test cases?">
    Sample test cases are always visible in the problem statement. Some problems may show additional test cases after you solve them.
    
    Hidden test cases remain secret to ensure solutions are general and not hardcoded.
  </Accordion>

  <Accordion title="What are interactive problems?">
    Interactive problems require your program to communicate with a judge program through standard input/output. Instead of reading all input at once, you:
    
    1. Read a query from the judge
    2. Respond with your answer
    3. Repeat until the problem is solved
    
    These problems test strategy and communication protocols.
  </Accordion>

  <Accordion title="What is Karel?">
    Karel is an educational programming environment using a robot in a grid world. Karel problems are perfect for beginners learning programming concepts:
    
    - **Commands**: `move()`, `turnleft()`, `pickbeeper()`, `putbeeper()`
    - **Conditions**: `frontIsClear()`, `nextToABeeper()`
    - **Loops**: `while`, `iterate`
    
    Karel helps you learn control flow, loops, and conditionals in a visual way.
  </Accordion>
</AccordionGroup>

## Tips for success

<Tip>
  **Start small**: Begin with easy problems to build confidence and learn the platform.
</Tip>

<Tip>
  **Read carefully**: Many wrong answers come from misreading problem constraints or output formats.
</Tip>

<Tip>
  **Test locally**: Run your code on sample inputs before submitting to catch obvious errors.
</Tip>

<Tip>
  **Learn from others**: After solving a problem, check editorials and other solutions to learn new techniques.
</Tip>

<Tip>
  **Practice regularly**: Consistency is key. Solve a few problems each week to steadily improve.
</Tip>
