## Overview

omegaUp's course management system enables educators to create comprehensive programming courses with multiple assignments, student enrollment, progress tracking, and automated grading. Courses provide a structured learning environment beyond individual contests.

## Courses vs contests

While contests and courses share some similarities, they serve different purposes:

| Feature | Contest | Course |
|---------|---------|--------|
| **Duration** | Fixed time window | Ongoing, multiple assignments |
| **Structure** | Single event | Multiple assignments/modules |
| **Enrollment** | Registration or invitation | Student enrollment with access codes |
| **Progress tracking** | Single scoreboard | Per-assignment progress and overall metrics |
| **Use case** | Competitions, assessments | Long-term learning, curriculum |
| **Grading** | Immediate competitive ranking | Cumulative progress tracking |

## Creating a course

<Steps>
  <Step title="Course setup">
    Define course name, description, start/end dates, and visibility settings
    
    ```json
    {
      "alias": "intro-to-algorithms",
      "name": "Introduction to Algorithms",
      "description": "Comprehensive course covering fundamental algorithms and data structures",
      "start_time": "2024-01-15T00:00:00Z",
      "finish_time": "2024-06-30T23:59:59Z",
      "is_public": true,
      "show_scoreboard": true
    }
    ```
  </Step>
  
  <Step title="Add assignments">
    Create assignments (similar to contests) with problems, deadlines, and point values
  </Step>
  
  <Step title="Enroll students">
    Add students individually, in bulk, or via registration codes
  </Step>
  
  <Step title="Monitor progress">
    Track student submissions, scores, and completion rates
  </Step>
  
  <Step title="Issue grades">
    Review final scores and export grade reports
  </Step>
</Steps>

## Assignment types

Courses support different assignment formats:

<AccordionGroup>
  <Accordion title="Homework assignments">
    Extended deadlines (days to weeks) for practice and learning. Students can attempt problems multiple times with instant feedback.
  </Accordion>

  <Accordion title="Exams">
    Time-limited assessments with restricted submission counts. Can enable/disable scoreboard visibility during exam periods.
  </Accordion>

  <Accordion title="Practice sets">
    Optional problem sets for additional practice. May not count toward final grade or have relaxed deadlines.
  </Accordion>

  <Accordion title="Projects">
    Long-term assignments with complex problems or multi-stage challenges.
  </Accordion>
</AccordionGroup>

## Student enrollment

### Enrollment methods

<Tabs>
  <Tab title="Individual enrollment">
    Manually add students one at a time using their omegaUp usernames or emails.
    
    ```php
    CourseController::apiAddStudent([
        'course_alias' => 'intro-to-algorithms',
        'usernameOrEmail' => 'student@university.edu'
    ]);
    ```
  </Tab>

  <Tab title="Bulk enrollment">
    Upload CSV file with student information for batch enrollment.
    
    ```csv
    username,email,name
    student1,student1@university.edu,John Doe
    student2,student2@university.edu,Jane Smith
    ```
  </Tab>

  <Tab title="Registration codes">
    Generate unique codes that students can use to self-enroll.
    
    ```json
    {
      "registration_code": "ABC123XYZ",
      "valid_until": "2024-01-31T23:59:59Z",
      "max_uses": 50
    }
    ```
  </Tab>
</Tabs>

<Tip>
  Registration codes are ideal for large courses where you want students to self-enroll without manual administration.
</Tip>

## Progress tracking

### Student dashboard

Students see:

- **Overall course progress**: Percentage of completed assignments
- **Assignment list**: Upcoming, active, and past assignments
- **Current scores**: Points earned vs maximum possible
- **Problem status**: Which problems have been solved
- **Deadlines**: Visual indicators for approaching due dates

## Grading system

### Automatic grading

All assignments are automatically graded based on test case results:

- **Immediate feedback**: Students see results instantly (unless configured otherwise)
- **Partial credit**: Award points for partially correct solutions
- **Test case visibility**: Control which test cases students can see
- **Multiple attempts**: Allow students to improve their solutions

### Grade calculation

Final course grades can be calculated using different methods:

<Tabs>
  <Tab title="Total points">
    Sum of all assignment scores
    
    ```
    Final Grade = Σ(Assignment Points) / Total Possible Points
    ```
  </Tab>

  <Tab title="Weighted average">
    Different assignments have different weights
    
    ```
    Final Grade = Σ(Assignment Points × Weight) / Σ(Max Points × Weight)
    ```
  </Tab>

  <Tab title="Best N assignments">
    Count only the best performing assignments
    
    ```
    Final Grade = Sum of top N assignment scores / (N × Max Points)
    ```
  </Tab>
</Tabs>

## Best practices

### Course structure

- **Weekly assignments**: One assignment per week with 3-5 problems
- **Gradual difficulty**: Start with easier problems, increase complexity over time
- **Topic grouping**: Group related problems in the same assignment
- **Clear deadlines**: Set consistent deadline patterns (e.g., every Sunday at 11:59 PM)

## Course API

Manage courses programmatically:

```bash
# Create a course
curl https://omegaup.com/api/course/create/ \
  -H "Cookie: ouat=$AUTH_TOKEN" \
  -d alias=intro-algorithms \
  -d name="Introduction to Algorithms"

# Get student progress
curl https://omegaup.com/api/course/studentProgress/ \
  -H "Cookie: ouat=$AUTH_TOKEN" \
  -d course_alias=intro-algorithms \
  -d username=student1
```
