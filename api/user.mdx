---
title: User API
description: User authentication and profile management endpoints
---

The User API provides endpoints for user authentication, registration, and profile management.

## Login

`POST /api/user/login/`

Authenticates a user and returns an authentication token.

<ParamField body="usernameOrEmail" type="string" required>
  Username or email address
</ParamField>

<ParamField body="password" type="string" required>
  User password
</ParamField>

<ResponseField name="auth_token" type="string">
  Authentication token for API requests
</ResponseField>

## Create User

`POST /api/user/create/`

Registers a new user account.

<ParamField body="username" type="string" required>
  Desired username (3-20 characters, alphanumeric)
</ParamField>

<ParamField body="password" type="string" required>
  Strong password (minimum 8 characters)
</ParamField>

<ParamField body="email" type="string" required>
  Valid email address
</ParamField>

<ParamField body="name" type="string" optional>
  Full name
</ParamField>

<ParamField body="birth_date" type="timestamp" optional>
  Birth date (UNIX timestamp)
</ParamField>

<ParamField body="gender" type="enum" optional>
  Gender: `female`, `male`, `other`, `decline`
</ParamField>

<ParamField body="recaptcha" type="string" optional>
  reCAPTCHA response token (required if CAPTCHA validation enabled)
</ParamField>

<ParamField body="is_private" type="boolean" optional>
  Whether profile is private (default: false)
</ParamField>

<ParamField body="parent_email" type="string" optional>
  Parent email (required for users under 13)
</ParamField>

<ResponseField name="username" type="string">
  Created username
</ResponseField>

## Verify Email

`GET /api/user/verifyEmail/`

Verifies a user's email address using verification ID.

<ParamField query="id" type="string" required>
  Email verification ID (sent via email)
</ParamField>

<ResponseField name="status" type="string">
  Returns `ok` if verification successful
</ResponseField>

## Change Password

`POST /api/user/changePassword/`

Changes the user's password. Requires authentication.

<ParamField body="old_password" type="string" required>
  Current password
</ParamField>

<ParamField body="password" type="string" required>
  New password
</ParamField>

<ResponseField name="status" type="string">
  Returns `ok` if password changed successfully
</ResponseField>

## Password Requirements

Passwords must meet these criteria:
- Minimum 8 characters
- At least one uppercase letter
- At least one lowercase letter
- At least one number

## Authentication

After login, include the auth token in subsequent requests:

```bash
curl -H "Authorization: token YOUR_AUTH_TOKEN" \
  https://omegaup.com/api/user/profile/
```

## Example: User Registration

```bash
curl -X POST https://omegaup.com/api/user/create/ \
  -d '{
    "username": "johndoe",
    "password": "SecurePass123",
    "email": "john@example.com",
    "name": "John Doe"
  }'
```

## Example: User Login

```bash
curl -X POST https://omegaup.com/api/user/login/ \
  -d '{
    "usernameOrEmail": "johndoe",
    "password": "SecurePass123"
  }'
```

## Example: Get User Profile

```bash
curl "https://omegaup.com/api/user/profile/?username=johndoe"
```
